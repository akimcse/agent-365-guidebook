런타임 위협 탐지

Copilot Studio / Azure AI Foundry 에이전트의 런타임 위협 탐지·차단은 서로 다른 3개 계층에서 일어납니다 — ① Copilot Studio 네이티브, ② Prompt Shields → Defender for AI(모델 레벨), ③ Agent 365 · Defender XDR(에이전트 런타임). 계층마다 탐지 엔진 · 커넥터 · 시그널 흐름 · 표출 포털 · 라이선스가 다릅니다.

실시간 보호 vs 탐지·조사Security for AI의 두 국면 — 막는다(prevent) vs 찾아내고 조사한다(detect)

실시간 보호 (real-time protection) 탐지·조사 (detection & investigation)
역할 tool 실행 직전 위험 액션을 차단(prevent) 위협 발생 후 near-real-time 알림 생성·조사(detect)
동작 Default 규칙(전 에이전트 audit) / Custom 규칙(고신뢰 위협 block + 스코프) 알림 → 인시던트 상관 → Advanced Hunting(KQL)
지원 위협 유형 차단 규칙 4종(완결): Secret exfiltration · Malicious content propagation · Evasion techniques · Unsafe email domain (jailbreak·XPIA는 탐지만, 차단 규칙 타입 아님) 에이전트 단(Agent 365, 비완결): jailbreak·XPIA·malicious content·secret leak·evasion·suspicious access + "and more" · 모델 단(MDC): 17~18종(아래 접기)

왜 Copilot Studio jailbreak는 Defender XDR 알림에 안 뜨나

CS 에이전트의 jailbreak가 실제로 차단(ContentFiltered)되고 Purview DSPM엔 기록되어도 Defender XDR 알림엔 안 뜰 수 있습니다 — 아래 세 경로가 모두 CS를 비켜가기 때문입니다. (반면 Foundry/Azure OpenAI 모델은 MDC 경로로 정상 표출)

① MDC 경로 ✗

모델 소유권
CS는 Microsoft 관리 모델(GPT·Claude 등, 고객 Azure 리소스 아님)을 사용. Defender for AI Services(MDC)는 고객 구독 내 Azure OpenAI/Foundry 리소스만 모니터링하므로 CS 모델은 감시 대상 밖 → AI.Azure_Jailbreak.* 경로를 애초에 안 탐.

② Agent 365 경로 ✗

Prompt Shields behavior 미지원
Prompt Shields 블록 이벤트는 Foundry·M365 Copilot Agent Builder만 BehaviorInfo에 기록되고 Copilot Studio는 아직 미지원. (CS는 콘텐츠가 아니라 tool invocation 평가로만 커버)

③ 차단규칙 억제

alert 대신 behavior
차단 규칙이 커버하는 에이전트는 near-real-time alert가 생성되지 않고 BehaviorInfo behavior로만 기록됩니다.

→ CS jailbreak는 CS 내장 모더레이션이 차단 + Purview에 기록되지만, MDC·Agent 365·차단규칙 세 경로 모두 비켜가 Defender 알림엔 안 뜨는 게 현재 사양입니다.

① 데이터 소스 연결 (Get started)

Agent 365 온보딩 시 security for AI 자동 활성화. 체크리스트 3종을 연결하면 검색·태세·탐지가 켜집니다.

Agent 365RequiredAI 에이전트 통합 가시성 · 온보딩 시 자동 완료
M365 커넥터Required조사 + Advanced Hunting 활성화(Entra ID Mgmt events · M365 activities). 연결돼야 alert·incident가 Defender 포털에 표출
Copilot StudioRequiredCS 에이전트 tool invocation 모니터·탐지·차단. 실시간 보호 토글 On + Power Platform 관리자 협업(App ID 교환)

🔑 M365 커넥터 = linchpin(CS 경로 한정): 미연결 시 CS 차단은 작동하나 alert·incident 미표출. Foundry/Prompt Shields(MDC) 알림은 커넥터 없이도 MDC→XDR 자동 통합으로 표출(전제: Azure 구독 + DfC 플랜).

② 실시간 보호 규칙 (Policies & rules)

온보딩은 탐지·감사까지. 실행 전 차단하려면 규칙 정의 필요(Create rule → Scope/Exclude/Detection types → Review).

  • Default 규칙 (audit): 전 에이전트 대상, 차단 없이 BehaviorInfo에 기록만 → 차단 전 가시성 확보. audit 모드에선 alert 표출.
  • Custom 규칙 (block): 매칭 액션 실행 전 차단 + behavior 기록, 에이전트 scope. 차단 규칙 커버 시 near-real-time alert 미생성(차단 우선).

③ Prompt evidence collection (기본 On)

각 알림에 탐지 유발 프롬프트 스니펫을 증거로 포함해 triage·근본원인 분석을 가속. 의심·보안 관련 부분만 포함, 민감정보·시크릿은 redact. 대화가 민감할 수 있어 정책에 따라 Off 가능.
